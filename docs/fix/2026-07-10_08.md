# Fix: dhash/ahash 感知哈希死计算

## 0. 改的代码大致位置
`books_of_time/media/hasher.py` — `inspect_image()` 方法
`books_of_time/media/downloader.py` — `fetch_media_source()` 方法
`tests/test_media_hasher.py` — 感知哈希测试
`tests/test_media_downloader.py` — 下载器集成测试

## 1. 为什么改
每次图片下载都计算三种感知哈希(pHash、dHash、aHash)，但只有 pHash 被查询和使用（`MediaSimilarityAnalyzer.analyze_phash` 只读取 `MediaAsset.phash`）。dHash 和 aHash 消耗 CPU 时间，写入 DB 后永不被读取。Schema 列(`dhash`/`ahash`)保留以备后用，但不应在每次下载时计算死值。

## 2. 改完预期效果如何
- 每次图片下载减少 2 次感知哈希计算(~50% CPU 节省)
- `MediaAsset.dhash` 和 `MediaAsset.ahash` 列写入 `NULL`，schema 不变，后续可随时恢复计算
- `MediaSimilarityAnalyzer` 不受影响（只使用 `phash`）
- `MediaService._record_media_state_events` 使用 `phash_hamming` distance，同样不受影响

## 3. 如何改的/改进思路
1. `hasher.py`：将 `_dhash(image)` / `_ahash(image)` 替换为 `None`（保留函数定义以便将来恢复）
2. `downloader.py`：移除 `dhash=image_metadata.dhash` / `ahash=image_metadata.ahash` 传参（默认 `None`）
3. 测试更新：`test_media_hasher` 和 `test_media_downloader` 断言改为 `is None` 而非 `isinstance(..., int)`
