# Fix: _enqueue_followup 缺少 idempotency_key

## 0. 改的代码大致位置
`books_of_time/collectors/latest_comments.py` — `_enqueue_followup()` 方法

## 1. 为什么改
`_enqueue_followup` 每次暂停时创建一个新的 `FETCH_LATEST_COMMENTS` 任务，但没有传 `idempotency_key`。如果 `max_scan_seconds` 设置过短，每次执行都会创建新的 follow-up 任务，导致队列爆炸（同一视频可能有数十个 follow-up 任务）。

## 2. 改完预期效果如何
同一视频的 follow-up 任务使用固定 key `fetxh_latest_comments:video:{bvid}:latest_followup`，`CollectionTaskRepository.enqueue` 自动去重：如果已有活跃 follow-up 任务，复用现有任务而非创建新任务。

## 3. 如何改的/改进思路
在 `_enqueue_followup` 调用 `enqueue` 时传入：
```python
idempotency_key=(
    f"{TaskKind.FETCH_LATEST_COMMENTS.value}:"
    f"{task.target_type}:{task.target_id}:latest_followup"
)
```
