# Fix: latest_comments raw_payloads count 无范围限制

## 0. 改的代码大致位置
`books_of_time/collectors/latest_comments.py` — `collect()` 方法和 `_count_raw_payloads()` 方法

## 1. 为什么改
`_count_raw_payloads` 使用 `SELECT COUNT(*) FROM raw_payloads`（无 WHERE 子句），统计整个数据库中所有 `raw_payloads` 记录。在多任务并发场景下，其他任务插入的 `raw_payloads` 会污染本任务的 `raw_payloads_saved` 统计。即使单 worker 场景下也依赖全局计数相减，语义不清晰。

`hot_comments.py` 采用局部计数器方案，`latest_comments.py` 应保持一致。

## 2. 改完预期效果如何
`raw_payloads_saved` 只统计本次 `collect()` 调用期间创建的 `RawPayload`（通过 `created_at >= payload_cutoff` 过滤），不受历史数据或其他并发任务的影响。

## 3. 如何改的/改进思路
1. 在 `_resolve_aid` 之前设置 `payload_cutoff = datetime.now(UTC)`
2. 删除 `raw_payloads_before = await self._count_raw_payloads(session)` 行
3. 将后方计数改为 scoped 查询：
   ```python
   raw_payloads_after = await session.scalar(
       select(func.count(RawPayload.id)).where(
           RawPayload.created_at >= payload_cutoff,
       )
   )
   ```
4. `raw_payloads_saved` 直接用 `int(raw_payloads_after or 0)` 而非前后差值
5. 删除无用的 `_count_raw_payloads` 方法
