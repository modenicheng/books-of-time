# Fix: 未知 cohort tier 配置被静默忽略

## 0. 改的代码大致位置

- `books_of_time/domain/cohort_policy.py`
- `tests/test_cohort_policy_config.py`
- `docs/CONFIGURATION.md`

## 1. 为什么改

`CohortPolicy.from_config()` 原实现只遍历内置的 `s/a/b` threshold 和
`s/a/b/c` interval 默认项，没有检查配置映射中是否残留未知键。因此
`tier_intervals_minutes.invalid` 或拼错的 `tier_policy` tier 会被静默忽略，并使用
默认采集频率继续运行。

这违反了 cohort 设计中“invalid tier 必须拒绝”的约束。对以采集完整性为中心的
服务，静默回退比启动时明确报错更危险：operator 可能以为已提高或降低某层采集
频率，但实际请求策略完全没有变化。

## 2. 改完预期效果如何

- `tier_policy` 只接受三个元配置键和 `s/a/b` threshold 键。
- `tier_intervals_minutes` 只接受 `s/a/b/c`。
- 任一未知键在 policy 解析时立即抛出包含键名的 `ValueError`。
- 合法的默认配置、部分覆盖和现有时间/评级/生命周期策略保持不变。

## 3. 如何改的/改进思路

在读取具体 threshold/interval 数值前，先计算输入键与允许键集合的差集，并按稳定
排序输出未知键。修复没有扩大顶层 C2 配置面，也没有提前校验 C3-C8 的独立执行
参数。两个回归用例先证明未知 interval tier 和未知 threshold tier 都不会报错，
再验证修复后两者被拒绝。最终执行全量测试和 Ruff。
