# Fix: 无 raw 的重复发现清空视频来源末次证据

## 0. 改的代码大致位置

- `books_of_time/db/repositories.py`
- `tests/test_discovery_scheduler.py`

## 1. 为什么改

`KnownVideoSourceRepository.upsert_for_video()` 原先在重复发现来源时无条件把
`last_raw_page_id` 写成当前参数。正式 `UserVideosCollector` 会传入 discovery raw
page ID，但诊断 `DiscoveryLoop` 不创建 raw page，调用时该参数为 NULL。只要在正式
采集后运行一次诊断 loop，已有的末次 raw provenance 就会被 NULL 覆盖，导致来源到
raw page/raw payload 的最近证据链永久丢失。

## 2. 改完预期效果如何

- 带 raw page 的正式发现继续推进 `last_raw_page_id`。
- 无 raw page 的诊断发现只推进 `last_seen_at`，不会清空已有 provenance。
- `first_raw_page_id` 始终保持首次证据，不受重复发现影响。
- 同一 BVID 的 stats task 幂等和多来源归属行为保持不变。

## 3. 如何改的/改进思路

重复来源 upsert 仅在 `raw_page_observation_id is not None` 时更新
`last_raw_page_id`。回归测试依次模拟首次正式发现、第二次正式发现和第三次无 raw
诊断发现，验证 last seen 前进到第三次，而 first/last raw page 仍指向已有证据。
