# Fix: VideoSnapshotScheduler 跳过不可用视频

## 0. 改的代码大致位置
`books_of_time/task_orchestrator/video_snapshot_scheduler.py`

## 1. 为什么改
`VideoSnapshotScheduler` 在安排视频快照时不检查视频可用性状态。如果视频已被删除、设为不可见或隐藏，scheduler 仍然会为其创建 `FETCH_VIDEO_STATS` 任务，导致：
- 给已删除视频安排无意义的采集任务
- 浪费 worker 循环和请求配额
- 日志中出现大量无意义的 404/权限错误

## 2. 改完预期效果如何
- `schedule_next_for_video`：如果该视频最新的 `VideoAvailabilitySnapshot` 状态不是 `visible`，直接返回 `None`，不安排快照。
- `schedule_terminal_snapshots`：遍历 `KnownVideo` 时，跳过不可用视频，只给可见视频安排 daily terminal snapshot。
- 没有 availability 记录的视频默认可继续安排（`latest is None → return True`），不影响正常视频的采集流程。

## 3. 如何改的/改进思路
在 `VideoSnapshotScheduler` 中新增 `_is_video_available` 辅助方法：
```python
async def _is_video_available(session: AsyncSession, bvid: str) -> bool:
    latest = await session.scalar(
        select(VideoAvailabilitySnapshot)
        .where(VideoAvailabilitySnapshot.bvid == bvid)
        .order_by(VideoAvailabilitySnapshot.captured_at.desc())
        .limit(1)
    )
    if latest is None:
        return True
    return latest.status == "visible"
```
然后在两个调度方法中调用该辅助方法过滤不可用视频。
