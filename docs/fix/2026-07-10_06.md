# Fix: BooksOfTimeBiliAPIClient.request 可变默认参数

## 0. 改的代码大致位置
`books_of_time/platforms/bilibili/request_client.py` — `BooksOfTimeBiliAPIClient.request()` 方法签名和参数传递

## 1. 为什么改
```python
params: dict[str, Any] = {},
data: dict[str, Any] | str | bytes = {},
files: dict[str, BiliAPIFile] = {},
headers: dict[str, str] = {},
cookies: dict[str, str] = {},
```
Python 可变默认参数在函数定义时只计算一次，后续所有调用共享同一个 dict 实例。如果上游 `bilibili_api` 库内部修改了这些 dict，状态会泄漏到后续请求中，导致字段污染、意外参数透传。

## 2. 改完预期效果如何
每次调用使用独立的局部变量 `_params`/`_data`/`_headers`/`_cookies`/`_files`，互不干扰。类型签名也更准确（`| None` 表示可选）。

## 3. 如何改的/改进思路
1. 参数默认值改为 `= None`（不可变）
2. 方法体内创建局部变量：
   ```python
   _params = params or {}
   _data: dict[str, Any] | str | bytes = {} if data is None else data
   _headers = headers or {}
   _cookies = cookies or {}
   _files = files or {}
   ```
3. 方法体内所有引用替换为局部变量（`params` → `_params` 等）
