# Fix: 普通应用启动提前加载 Alembic autogenerate 插件

## 0. 改的代码大致位置

- `books_of_time/db/migrations.py`
- `tests/test_schema_migrations.py`

## 1. 为什么改

`books_of_time.db.migrations` 在模块顶层导入 `Alembic Config` 和 `ScriptDirectory`。CLI 导入健康检查时会间接导入该模块，导致普通 worker、真实请求烟测和长期服务启动提前加载 `alembic.autogenerate.*` 插件，并输出多条与采集无关的插件初始化日志。

schema revision 只在 `service doctor`、`service health` 或服务启动检查时需要读取。普通 collector/client import 不应承担 Alembic autogenerate 的导入成本和日志噪声。

## 2. 改完预期效果如何

- 普通导入 `books_of_time.db.migrations` 不加载 `alembic.autogenerate.*`。
- 只有实际调用 `get_expected_schema_revision()` 时才导入 Alembic 脚本读取组件。
- schema revision 检查、migration 与 doctor 行为保持不变。

## 3. 如何改的/改进思路

将 `Config` 和 `ScriptDirectory` 移到 `get_expected_schema_revision()` 函数内部按需导入。新增独立 Python 子进程回归测试，检查导入 helper 后 `sys.modules` 中不存在 `alembic.autogenerate` 模块，避免测试进程既有 import 状态干扰结论。
