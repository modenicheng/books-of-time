# Fix: Worker loop 因单任务异常整体崩溃

## 0. 改的代码大致位置
`books_of_time/worker.py` — `run_once()` 失败分支末尾 (line 134)

## 1. 为什么改
`run_once` 在记录失败并提交 DB 后执行 `raise`（line 134），将异常重新抛出。`run_loop` 没有 `try/except` 包裹，导致单个 collector 异常直接终止整个 worker 进程，队列中所有待处理任务被放弃。

虽然设计文档注明 "If run_once() raises, run_loop() propagates the exception"，但实际运维场景下 worker 应具备容错能力——失败已记录到 DB（coverage stat + task status），不需要 crash 通知。

## 2. 改完预期效果如何
`run_once()` 失败后返回 `True`（表示一个任务被处理），`run_loop` 继续循环处理下一个任务。失败信息已通过 `CollectionCoverageStat` 和 `CollectionTask.status` 记录到 DB，后续可通过 CLI 查看。

## 3. 如何改的/改进思路
将 `raise`（line 134）替换为 `return True`。`run_loop` 中：
- 返回 `True` → executed_count += 1，continue（立即处理下一个任务）
- 返回 `False` → 进入 idle sleep

失败后返回 `True` 而非 `False` 是合理的：失败也是一个已处理的任务，不应进入 sleep。
